# Idea 25: Database Schema Evolution Impact Analysis

Take OSS projects with databases (Django apps, Rails apps, Supabase, Hasura) at a point where a schema migration is proposed. Task: identify all code paths affected by the migration — not just direct ORM queries, but views, serializers, API responses, cached values, background jobs, and data migrations that reference the affected tables/columns. Ground truth: the actual PR that accompanied the migration (all files changed). Unique: the 'code' is split between declarative schema (migrations) and imperative access (ORM calls scattered across the codebase). Understanding must bridge these two representation layers. Enterprise DB migrations are the #1 source of production incidents.
