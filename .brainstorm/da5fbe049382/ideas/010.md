# Idea 10: Permission and Access Control Audit

Take OSS projects with complex auth systems (Keycloak, Casbin, OPA, or apps using them like GitLab, Harbor). Task: given a specific user role, enumerate exactly what API endpoints/resources they can access. Ground truth extracted from the project's own RBAC tests and permission matrices. The structural novelty: understanding isn't about code flow — it's about POLICY evaluation. The agent must trace through policy definitions (RBAC YAML, Rego rules, Casbin models), middleware chains, and decorator/annotation patterns to build a complete access map. Enterprise security teams do this audit quarterly.
