# Idea 28: Tribal Knowledge Extraction via PR Review Comments

Take the PR review comment history of major OSS projects (Kubernetes, Linux kernel, Rust compiler — all have public reviews). Extract the UNWRITTEN RULES — conventions that are enforced by reviewers but not documented anywhere. E.g., 'we never use async in this module because of X', 'always check error codes from this syscall'. Agent reads N PRs with review feedback, then predicts what review comments a NEW PR would receive. Ground truth: actual review comments on held-out PRs. Unique: the 'codebase' being understood is the SOCIAL KNOWLEDGE embedded in review history, not just the code. Tests ability to extract implicit conventions from human discourse about code.
