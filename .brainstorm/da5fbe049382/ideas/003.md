# Idea 3: Living Documentation Sync Detector

Take OSS projects with extensive docs (Kubernetes, Terraform, Django) at specific historical commits where docs were known to be out-of-sync with code. The task: identify all documentation-code inconsistencies in a given module. Ground truth extracted from the actual doc-fix PRs that followed. This measures a workflow that happens constantly in enterprises — someone asks 'is the docs right?' and the answer requires understanding both the code semantics AND the doc claims. Scoring is precision/recall on identified inconsistencies. Unique structure: the agent must compare TWO representation systems (prose docs vs code behavior) rather than navigating one.
