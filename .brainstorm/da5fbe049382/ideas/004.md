# Idea 4: Configuration Drift Forensics

Use real OSS projects with multiple deployment configurations (Kubernetes Helm charts, Terraform modules, Ansible playbooks). Introduce realistic config drift: one env has a param that diverged from the template. Agent must find ALL drift points across a multi-service deployment config. Ground truth from actual config-fix PRs. The structural novelty: the codebase isn't code — it's declarative configuration. Understanding tools must parse YAML/HCL/JSON hierarchies, understand inheritance/override chains, and detect semantic (not just syntactic) differences. Enterprise #1 pain point. Scoring: drift-point precision/recall.
