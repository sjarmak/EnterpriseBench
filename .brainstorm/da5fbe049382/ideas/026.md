# Idea 26: Concurrency Bug Pattern Recognition

Collect real concurrency bugs from OSS projects (Go goroutine leaks, Rust unsafe races, Java deadlocks, Python GIL-related issues) that were found in production. Present the code at the buggy commit. Agent must identify: the race/deadlock/ordering bug, the specific interleaving that triggers it, and the affected shared state. Ground truth: the actual fix commit + the issue description. Unique structural property: these bugs are INVISIBLE to sequential code reading — they only manifest under specific execution orderings. The agent must reason about CONCURRENT state spaces, not sequential control flow. This is a different cognitive operation than any prior art measures.
