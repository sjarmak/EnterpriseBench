# Idea 23: Error Message Provenance Tracing

Users file bugs with error messages. Take real OSS issues where the bug report contains an error message or stack trace. The agent's task: given ONLY the error text (no stack trace pointing to files), find the exact code location that generates this message, the conditions that trigger it, and the state that led there. Ground truth: the actual fix PR's changed files. Unique: the starting point is a STRING, not a file or function. The agent must reverse-engineer from user-visible output back through string construction, error handling chains, and exception propagation. Tests backward-tracing ability that grep alone cannot solve (formatted strings, i18n, error codes).
