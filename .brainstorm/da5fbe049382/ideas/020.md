# Idea 20: Compliance Regulation Mapping

Take OSS projects that handle personal data (Sentry, Mattermost, GitLab, Supabase) and a regulation spec (GDPR articles, SOC2 controls, HIPAA requirements). Task: map each regulatory requirement to the specific code locations that implement (or fail to implement) it. E.g., 'GDPR Art 17 right to deletion' → which code paths handle user data deletion, are there any data stores that retain PII after deletion? Ground truth from the project's actual compliance documentation + security audits. Structural novelty: the 'query' is a LEGAL document, not a technical one. Understanding must bridge regulatory language to code implementation.
