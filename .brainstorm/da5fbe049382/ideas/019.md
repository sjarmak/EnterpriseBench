# Idea 19: Architecture Decision Record Reconstruction

Take OSS projects with ADRs (Architecture Decision Records) at a specific point in time. Remove the ADRs. Task: reconstruct the key architectural decisions by reading the codebase. The agent must infer WHY things are built the way they are — not just describe the architecture, but explain the tradeoffs that led to it. Ground truth: the actual ADRs. Scoring: coverage of decisions + correctness of rationale. This is fundamentally different from code navigation — it's INTENT RECONSTRUCTION from artifacts. It measures whether a tool can understand the reasoning embedded in code structure.
