# Idea 9: Incident Replay with Telemetry Streams

Reconstruct real incidents from OSS projects that have public postmortems (Kubernetes outage reports, GitLab incident reviews, Cloudflare blog posts). Provide the agent with: the codebase at the pre-incident commit, simulated telemetry (logs, metrics extracted from the postmortem narrative), and the alert that triggered investigation. Agent must produce: root cause, affected components, and remediation. Ground truth: the actual postmortem findings. Unique structure: the input isn't just code — it's code + time-series data + alert context. The agent must correlate telemetry patterns with code paths, which is how real incident response works.
