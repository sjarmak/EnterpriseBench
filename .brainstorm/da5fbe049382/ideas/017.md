# Idea 17: Regression Bisection via Semantic Diffing

Take OSS projects with known regressions that were hard to find (bugs that took 10+ commits to bisect). Provide the agent with the failing test + a range of 50-200 commits. Instead of running bisect mechanically, the agent must use SEMANTIC understanding of each commit's intent (from diffs + messages) to narrow to the causal commit in fewer steps than binary search. Scoring: number of commits examined before identifying the correct one. Lower is better. Ground truth: the actual revert commit. This measures whether codebase understanding tools enable smarter-than-mechanical debugging — can semantic comprehension beat O(log n)?
