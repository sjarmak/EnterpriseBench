# Idea 18: Support Ticket Triage and Code Mapping

Simulate customer support by providing real GitHub issues that were filed as vague bug reports (e.g., 'app crashes when I do X'). Agent must: (1) reproduce the issue conceptually by identifying the code path, (2) classify severity, (3) identify the owning team/module, (4) find related past issues. Ground truth: the actual issue labels, assignees, linked PRs, and duplicate markings from the project's issue tracker. Unique structure: input is NATURAL LANGUAGE from a non-developer (the customer), and the agent must bridge the semantic gap between user-facing behavior description and codebase internals. Enterprise support teams do this hundreds of times per day.
