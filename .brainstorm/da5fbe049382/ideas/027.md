# Idea 27: Build System Dependency Puzzle

Take projects with complex build systems (Bazel in TensorFlow/Chromium, CMake in LLVM/Qt, Gradle in Android). Introduce a change to a library target. Task: predict the exact set of build targets that will need rebuilding AND the order. Ground truth: the actual build system's output (bazel query, cmake --graphviz, gradle dependencies). Unique: the 'code' being understood is the BUILD SYSTEM itself — BUILD files, CMakeLists.txt, build.gradle — which is a meta-layer ABOUT the code, not the code. Understanding tools must parse a declarative dependency specification language and simulate its evaluation. This is orthogonal to understanding source code.
