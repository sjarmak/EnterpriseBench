# Idea 2: Git Archaeology Competitions

Tasks derived entirely from git history forensics on massive OSS repos (Linux kernel, Chromium, LLVM). Each task gives a symptom (regression, performance cliff, behavior change) and a date range. The agent must use git bisect-like reasoning, blame traversal, and commit message analysis to identify the causal commit(s). Ground truth is the actual revert/fix commit. No code needs to be written — the output is a structured attribution (commit SHA + reasoning chain). Measures temporal codebase understanding: can the tool navigate 20+ years of history across 50K+ commits? Enterprise codebases have this exact archaeology problem. Scoring: did you find the right commit, and was your reasoning chain valid (each step references real commits that logically narrow the search)?
