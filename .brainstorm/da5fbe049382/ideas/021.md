# Idea 21: Performance Cliff Prediction from Code Review

Take real OSS performance regressions that were caught in production (documented in blog posts, issue trackers). Present the PR that CAUSED the regression as a code review. Agent must identify: will this change cause a performance problem, and if so, what's the mechanism (O(n²) hidden in a loop, cache invalidation, lock contention, memory allocation pattern)? Ground truth: the actual performance regression report + fix. Unique: the agent sees ONLY the diff (like a code reviewer), not runtime data. Must predict dynamic behavior from static analysis. Measures whether understanding tools can catch what human reviewers missed.
