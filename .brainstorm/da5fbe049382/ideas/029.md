# Idea 29: Cross-Project Design Pattern Transfer

Take two OSS projects that solve the SAME problem differently (e.g., three HTTP routers: Gin vs Echo vs Chi in Go, or Express vs Koa vs Fastify in JS). Task: given a feature implemented in Project A, describe how to implement the equivalent in Project B, accounting for Project B's architecture and conventions. Ground truth: cases where the feature actually exists in both (compare implementations). Scoring: does the proposed approach match Project B's actual patterns? Unique structure: requires understanding TWO codebases' architectural philosophies and TRANSLATING between them — not porting code, but porting design intent. Enterprise teams evaluating framework switches need this.
