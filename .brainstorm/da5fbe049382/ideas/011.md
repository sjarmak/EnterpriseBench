# Idea 11: Test Gap Cartography

Given a large OSS project with existing tests, identify which code paths have NO test coverage and rank them by risk. Agent produces a structured map: untested function → risk level → suggested test type. Ground truth: extract from real coverage reports (lcov, coverage.py) + actual test additions in subsequent PRs. The structural twist: this requires understanding both what the code DOES (to assess risk) and what the tests COVER (to find gaps). Two codebases in one — the production code and the test suite — must be understood in relation to each other. Enterprise teams with 'increase coverage' mandates do exactly this.
