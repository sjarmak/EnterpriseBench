# Idea 15: CI Pipeline Optimization Oracle

Take real OSS projects with complex CI (Chromium, Rust compiler, LLVM, PyTorch) that have pipeline config + build time data in public CI logs. Task: given the CI configuration and historical timing data, identify the optimal parallelization strategy, unnecessary steps, and bottleneck stages. Ground truth: actual CI optimization PRs from the project. The agent must understand both the BUILD system (CMake, Bazel, Cargo) and the CI system (GitHub Actions, Jenkins, BuildKite) and how they interact. Scoring on predicted speedup vs actual achieved speedup. Enterprise teams spend huge effort on CI optimization — this measures understanding of build/test dependency DAGs.
