# Idea 1: Dependency Graph Traversal Races

Instead of task-based evaluation, the benchmark IS the dependency graph. Take the top 50 most-depended-on OSS packages (lodash, requests, protobuf, grpc, etc.) and their real transitive dependency trees. The task: given a CVE injected at a leaf dependency, trace the blast radius to all affected consumers. Scoring is graph coverage — what percentage of the actual affected path did the agent discover? No code generation, no patches. Pure graph-aware codebase traversal. The 'enterprise condition' is that real dependency trees are 500+ nodes deep with version conflicts, optional deps, and platform-specific branches. Tools that understand package manifests, lock files, and import graphs score higher than tools doing grep.
