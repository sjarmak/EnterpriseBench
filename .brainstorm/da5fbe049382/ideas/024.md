# Idea 24: Adversarial Codebase Sabotage Detection

Plant subtle, realistic bugs into real OSS codebases (mimicking actual vulnerabilities: off-by-one, race conditions, unchecked null, SQL injection through ORM bypass). The agent performs a security audit — not of the REAL code, but of the sabotaged version. Ground truth: the planted defects + their locations. Scoring: detection rate (recall) and false positive rate. The bugs are designed to be invisible to grep/pattern matching — they exploit SEMANTIC understanding gaps (e.g., a correct-looking function that fails only on edge cases). This inverts the normal benchmark: instead of finding what's MISSING, find what's WRONG. Measures defensive security audit capability.
