# Idea 14: Cross-Language FFI Bridge Comprehension

OSS projects with FFI boundaries: CPython (C↔Python), Node.js native addons (C++↔JS), PyO3/maturin (Rust↔Python), JNI in Android. Task: trace a user-visible behavior through the language boundary. E.g., 'what happens when you call os.listdir() in CPython?' — trace from Python through C API to POSIX. Ground truth: the actual call chain with function names on both sides. Unique structure: understanding must cross LANGUAGE boundaries where naming conventions, memory models, and calling conventions all change. The tool must parse two completely different AST structures and connect them through binding declarations. Enterprise codebases with native extensions face this constantly.
