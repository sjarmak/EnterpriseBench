# Idea 16: Ecosystem Health Dashboard Construction

Instead of per-task evaluation, the benchmark is a SYNTHESIS task. Give the agent the top 20 packages in a language ecosystem (e.g., Python data science: numpy, pandas, scikit-learn, scipy, matplotlib...) and ask it to produce a health report: maintenance status, bus factor, API stability, breaking change trajectory, security posture. Ground truth assembled from GitHub stats, CVE databases, and actual ecosystem health reports (e.g., Tidelift, Snyk State of Open Source). Unique: this tests BREADTH of understanding across many codebases simultaneously rather than DEPTH in one. Enterprise architecture teams must evaluate ecosystem risk exactly like this.
